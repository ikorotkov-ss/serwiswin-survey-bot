# История проекта / Best Practices

## 03.06.2026 — Развёртывание Survey Bot на VPS (1gb.ru)

### Задача
Запустить Telegram-бота для сбора опросов сотрудников на удалённом сервере, чтобы он работал 24/7.

### Выбор хостинга
- **Рассматривали:** Reg.ru (Free Tier 6 мес.), Beget (был shared-хостинг, не подошёл), 1gb.ru
- **Выбрали:** 1gb.ru — тестовый период 10 дней (бесплатно)
- **Почему не Reg.ru:** не успели разобраться с интерфейсом, 1gb.ru оказался проще
- **Дата-центр:** Франкфурт (для доступа к Telegram API; Москва — риск блокировок)
- **ОС:** Ubuntu 22.04 LTS (стабильнее 24.04 для пакетов)

### Процесс настройки
1. Создали VPS через панель 1gb.ru
2. Ждали 20-30 мин. создания сервера
3. Web SSH в браузере работал плохо — зависал
4. Проблема: пароль скрыт точками, не скопировать
5. **Решение:** через Web SSH выполнили `passwd`, задали новый пароль
6. Подключились через SSH с Mac (мой инструмент)
7. Установили: Python, ffmpeg, whisper-cli (сборка из исходников через cmake)
8. Скачали модель ggml-small.bin (465MB) с HuggingFace
9. Код скопировали через scp (не git clone — не было SSH-ключа на GitHub)

### Проблемы и решения
| Проблема | Решение |
|----------|---------|
| GitHub не пускает git clone (нет SSH-ключа) | Копировать через scp с паролем |
| whisper-cli не собирается через `make whisper-cli` | Новая версия использует cmake |
| Ansible не работает с паролем без sshpass | Установили sshpass через brew |
| БД хранилась в папке с кодом | Вынесли в `/var/lib/survey-bot/` через DATA_DIR |

### Архитектура
- **Код:** `/opt/survey-bot/` — только код из репозитория
- **Данные:** `/var/lib/survey-bot/` — БД, аудио, модель
- **Автозапуск:** systemd-сервис (restart=always)
- **Бэкапы:** cron ежедневно в 4:03

### Best Practices (на будущее)
1. **Всегда разделять код и данные** — DATA_DIR env var
2. **One-command deploy** — `setup.sh <IP> <TOKEN> <ADMIN_IDS>`
3. **Ansible плейбук** — для полной автоматизации
4. **Не использовать Web SSH** — только нативный SSH-клиент
5. **Frankfurt для Telegram** — Москва блокирует
6. **Ubuntu 22.04** — проверенная стабильность
7. **sshpass для Ansible с паролем** — или настраивать SSH-ключи
8. **Домен не нужен** для Telegram-бота

### Структура deploy/
```
deploy/
├── setup.sh                  # Одна команда для новичка
├── playbook.yml              # Ansible (внутренности)
├── survey-bot.service.j2     # systemd с DATA_DIR
├── inventory.example         # Пример подключения
└── README.md                 # Полная инструкция
```
